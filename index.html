Two Pointers
Foundation Technique
54 problems · sorted array, linked list, string
Why It Matters: Tests ability to eliminate search space using invariants. O(n) solutions to O(n²) problems demonstrate insight, not brute force.
Core Idea: Maintain lo/hi (or slow/fast) pointers. Decide which to move based on a monotone condition. The invariant — what you prove about the space between pointers — IS the algorithm.

Binary Search
Foundation Technique
46 problems · classic search, rotated, answer space, matrix
Why It Matters: Binary search is the canonical O(log n) technique. Every FAANG interview expects it on sorted arrays, rotated arrays, and — critically — on the answer space itself (binary search on value, not index).
Core Idea: Maintain invariant: answer is always in [lo, hi]. Choose mid; decide which half to discard based on a monotone predicate. Template: lo=0, hi=n-1; while(lo≤hi); always shrink the search space.

Sliding Window
Core DSA Technique
41 problems · subarray, substring, frequency maps, deque variants
Why It Matters: Converts O(n²) brute-force subarray enumeration into O(n) single-pass solutions by maintaining a live window [left, right] updated incrementally. The "aha" of recognizing when the window should shrink is a genuine test of algorithmic intuition.
Core Idea: The window always represents a valid/optimal range. Instead of recomputing from scratch, add one element at right and remove one at left. The invariant is what makes the window valid — if you can't define it, the technique doesn't apply.

Red Flags
🚩 Negatives in sum problem — The shrink invariant breaks; negative values destroy monotonic property. Use prefix-sum + HashMap instead.
🚩 Not updating maxFreq correctly — In #424, maxFreq is intentionally NOT decreased when shrinking. Decreasing it leads to wrong answer.
🚩 Off-by-one in fixed window — After init loop, right should be k-1, not k. Always verify first window indices.
🚩 Forgetting to remove outgoing element — When left advances, decrement count of nums[left] and delete key if count reaches 0.
🚩 Directly solving "exactly K" with single window — Writing if distinct == k: shrink is wrong. Always use exactly(K) = atMost(K) − atMost(K-1).
